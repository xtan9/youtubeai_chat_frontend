# Shadow tokens

Tailwind 4 provides a complete drop-shadow scale plus an inset variant.
We adopt the defaults as our system contract.

## Scale

| Token | Utility | Value | When to reach for |
|-------|---------|-------|-------------------|
| `--shadow-2xs` | `shadow-2xs` | `0 1px rgb(0 0 0 / 0.05)` | Hairline rest state, separators |
| `--shadow-xs` | `shadow-xs` | `0 1px 2px 0 rgb(0 0 0 / 0.05)` | Subtle lift (rest button, chip) |
| `--shadow-sm` | `shadow-sm` | `0 1px 3px 0 rgb(0 0 0 / 0.1), 0 1px 2px -1px rgb(0 0 0 / 0.1)` | Default card |
| `--shadow-md` | `shadow-md` | `0 4px 6px -1px rgb(0 0 0 / 0.1), 0 2px 4px -2px rgb(0 0 0 / 0.1)` | Hover state, raised card |
| `--shadow-lg` | `shadow-lg` | `0 10px 15px -3px rgb(0 0 0 / 0.1), 0 4px 6px -4px rgb(0 0 0 / 0.1)` | Floating panel (popover, dropdown) |
| `--shadow-xl` | `shadow-xl` | `0 20px 25px -5px rgb(0 0 0 / 0.1), 0 8px 10px -6px rgb(0 0 0 / 0.1)` | Modal, dialog |
| `--shadow-2xl` | `shadow-2xl` | `0 25px 50px -12px rgb(0 0 0 / 0.25)` | Hero callout, full-screen modal |
| `--shadow-inner` | `shadow-inner` | `inset 0 2px 4px 0 rgb(0 0 0 / 0.05)` | Pressed-state interior (dense form well) |

## Colored shadows

For the brand purple glow on the home Summarize button (and similar
emphasized CTAs), Tailwind's color-prefixed variants apply tinted shadow
colors:

```tsx
<button className="shadow-lg shadow-purple-500/25 hover:shadow-purple-500/40">
  Summarize
</button>
```

The `shadow-purple-500/25` modifier reads from the Tailwind 4 default
`--color-purple-500` palette. PRs 2-6 may formalize this as a
`shadow-brand-glow` utility once enough cluster components share the
pattern.

## Drop-shadow filter

For SVG icons and image cutouts, prefer the `drop-shadow-*` utilities
(filter-based, follows the alpha channel). Standard Tailwind 4
`drop-shadow-{xs,sm,md,lg,xl,2xl}` is available out of the box.
