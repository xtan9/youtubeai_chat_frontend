# Spacing tokens

Tailwind 4 uses a single base unit, `--spacing: 0.25rem` (4px), and
**multiplies** it for every spacing utility. This is the canonical 4-px
grid: every padding, margin, gap, width, and height utility comes off of
the same base.

```css
/* From Tailwind 4 defaults */
--spacing: 0.25rem;
```

## Mapping

| Utility | Multiplier | Computed | Pixels |
|---------|-----------|----------|--------|
| `p-0` / `m-0` / `gap-0` | 0 | 0 | 0 |
| `p-px` | (special) | 1px | 1 |
| `p-0.5` | 0.5× | 0.125rem | 2 |
| `p-1` | 1× | 0.25rem | 4 |
| `p-1.5` | 1.5× | 0.375rem | 6 |
| `p-2` | 2× | 0.5rem | 8 |
| `p-3` | 3× | 0.75rem | 12 |
| `p-4` | 4× | 1rem | 16 |
| `p-5` | 5× | 1.25rem | 20 |
| `p-6` | 6× | 1.5rem | 24 |
| `p-8` | 8× | 2rem | 32 |
| `p-10` | 10× | 2.5rem | 40 |
| `p-12` | 12× | 3rem | 48 |
| `p-16` | 16× | 4rem | 64 |
| `p-20` | 20× | 5rem | 80 |
| `p-24` | 24× | 6rem | 96 |
| `p-32` | 32× | 8rem | 128 |

The same multipliers apply to every spacing-aware utility: `m-*`,
`px-*`, `mt-*`, `gap-*`, `space-x-*`, `w-*`, `h-*`, `inset-*`, `top-*`,
`-mt-*` (negative), etc.

## When to reach for arbitrary values

Almost never. The 4-px grid is the foundation; if a design calls for
`p-3.5` (14px, half-step), use `p-3.5`. Only escape to `p-[13px]` if
absolutely required, and document why in a code comment.

## Deviation from the original plan

The B plan §4 originally specified `--space-px`, `--space-1`,
`--space-2`, …, `--space-32` as discrete `@theme` tokens. Research into
Tailwind 4's `@theme` directive (see B PR 1 step 2) revealed Tailwind 4
uses a **single `--spacing` base unit** that multiplies for every
utility — discrete `--space-*` tokens are not the v4 idiom and would
fight the framework. We adopt the v4 single-base pattern instead. Net
effect on consumer code: identical (the same `.p-4`/`.m-8`/etc. classes
work).
